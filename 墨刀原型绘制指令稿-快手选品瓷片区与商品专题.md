# 墨刀原型绘制指令稿：快手选品瓷片区与商品专题

## 1. 绘制目标

基于《快手选品瓷片区与商品专题需求说明》，在墨刀中绘制一套可用于产品评审、研发理解和后续视觉细化的中高保真原型。

本次原型重点表达：

1. 小程序端快手选品首页新增运营瓷片区。
2. 小程序端商品专题列表页和商品专题详情页。
3. 管理后台运营瓷片配置、商品专题管理、专题分类管理、专题货盘管理。
4. 关键交互、空状态、异常状态、弹窗/抽屉/导入结果反馈。

本次原型不表达：

1. 不新增快手首页金刚区专题入口。
2. 不改造、不优化、不重新绘制通用 Banner 广告位配置能力。
3. 不设计 H5 跳转能力。
4. 不重新设计快手商品卡片组件，专题详情商品列表直接复用现有快手商品卡片组件形态。

## 2. 画布与命名

### 2.1 画布分组

请在墨刀中按系统拆分页面分组：

| 分组 | 页面类型 | 建议画布命名 |
| --- | --- | --- |
| 小程序端 | 移动端页面 | 小程序-快手选品首页、小程序-专题列表页、小程序-专题详情页 |
| 管理后台 | Web 后台页面 | 后台-运营瓷片配置、后台-商品专题管理、后台-专题新增编辑 |
| 弹窗/抽屉 | 后台弹层 | 弹窗-主推资源位配置、弹窗-瓷片内容配置、弹窗-瓷片点击明细、抽屉-专题分类管理、弹窗-商品选择、弹窗-Excel导入结果 |
| 状态页 | 空/异常状态 | 状态-小程序专题列表、状态-小程序专题详情、状态-后台列表与表单 |

### 2.2 设计规格

| 端 | 建议尺寸 | 说明 |
| --- | --- | --- |
| 小程序 | 375 x 812 或等比例手机尺寸 | 保持接近现有快手选品页面风格 |
| 管理后台 | 1440 x 900 | 使用 Vue3 + Arco Design 后台风格 |

小程序页面需要尽量贴近当前截图中的视觉密度：顶部区域、圆角容器、商品卡片、筛选项和底部导航可参考现有快手选品风格。

管理后台页面需要按 Arco Design 常见后台结构绘制：页面标题、筛选表单、按钮组、表格、分页、弹窗、抽屉、表单校验提示。

## 3. 小程序端页面

### 3.1 小程序-快手选品首页

#### 页面目标

在现有快手选品首页中，于金刚区下方、商品分类 Tab 上方新增运营瓷片区。

#### 页面结构

```text
现有顶部区域
搜索框
现有金刚区
运营瓷片区
商品分类 Tab
筛选排序区
商品列表
底部导航
```

#### 运营瓷片区布局

绘制固定 1 大 + 4 小结构：

```text
┌──────────────┐  ┌──────────┐ ┌──────────┐
│              │  │ 小瓷片 1  │ │ 小瓷片 2  │
│   大瓷片      │  └──────────┘ └──────────┘
│              │  ┌──────────┐ ┌──────────┐
│              │  │ 小瓷片 3  │ │ 小瓷片 4  │
└──────────────┘  └──────────┘ └──────────┘
```

#### 内容标注

| 区域 | 展示内容 | 原型标注 |
| --- | --- | --- |
| 大瓷片 | 主推资源位封面图 | 标注“支持 1-3 条内容，按排序轮播” |
| 小瓷片 1-4 | 次级资源位封面图 | 标注“每个位置固定 1 条内容” |

#### 交互标注

1. 点击任意瓷片，先记录瓷片点击埋点，再按配置跳转。
2. 跳转类型为商品专题时，进入专题详情页；未登录时先登录，登录成功后回到目标专题详情。
3. 跳转类型为小程序路由时，跳转到指定 route。
4. 从瓷片进入专题后，专题访问和专题内商品行为需要携带来源瓷片内容 ID。

#### 状态

| 状态 | 原型表达 |
| --- | --- |
| 正常态 | 展示完整 1 大 + 4 小瓷片区 |
| 接口失败 | 隐藏瓷片区，首页其他内容正常展示，并标注“不阻塞首页主流程” |
| 图片加载失败 | 单个瓷片展示默认占位图 |
| 跳转专题失效 | Toast：内容已失效/专题已下架 |

### 3.2 小程序-商品专题列表页

#### 页面目标

展示已上架商品专题集合。未登录用户允许浏览列表，进入详情时需要登录。

#### 入口说明

专题列表页可由运营复用现有通用 Banner 配置能力跳转进入。本需求不绘制通用 Banner 配置功能，也不改造 Banner 功能。

#### 页面结构

```text
顶部导航
专题头图
专题分类筛选区
专题列表
```

#### 顶部导航

| 内容 | 说明 |
| --- | --- |
| 返回按钮 | 返回上一页；无上一页时返回快手选品首页 |
| 标题 | 商品专题 |

#### 专题头图

| 内容 | 说明 |
| --- | --- |
| 专题头图 | 使用专题创建/编辑时配置的专题封面/头图，不使用通用 Banner 广告位 |

#### 专题分类筛选区

专题分类区域按筛选项绘制，不是频道 Tab。

交互要求：

1. 后台没有配置专题分类时，隐藏整个分类筛选区。
2. 默认不选中任何分类，展示全部已上架专题。
3. 分类单选。
4. 点击一个分类后，列表只展示该分类下专题。
5. 已选中分类再次点击，取消选中，恢复展示全部已上架专题。
6. 未关联分类的专题在默认未筛选状态下展示；选中分类后不展示未分类专题。

#### 专题列表

专题卡片内容：

| 内容 | 说明 |
| --- | --- |
| 专题封面 | 后台配置 |
| 专题名称 | 后台配置 |
| 专题副标题/描述 | 后台配置，可为空 |
| 商品预览 | 展示专题内佣金率倒序 Top 3 商品 |
| 预览商品信息 | 商品主图、价格、佣金率、佣金金额 |

交互要求：

1. 点击专题卡片，已登录用户进入专题详情页。
2. 未登录用户点击专题卡片时，触发登录；登录成功后进入原目标专题详情页。
3. 列表仅展示已上架专题。
4. 列表按后台排序展示，同排序下可按更新时间倒序。

#### 状态

| 状态 | 原型表达 |
| --- | --- |
| 无专题分类 | 隐藏分类筛选区 |
| 全部无专题 | 空状态：暂无专题 |
| 当前分类无专题 | 空状态：当前分类暂无专题 |
| 专题列表接口失败 | 错误状态：加载失败，下拉刷新或点击重试 |
| 专题封面加载失败 | 展示默认专题封面 |

### 3.3 小程序-商品专题详情页

#### 页面目标

展示某个专题下的商品货盘，支持专题内搜索、排序、批量添加货架和商品卡片操作。

#### 访问规则

1. 专题详情页需要登录访问。
2. 未登录用户从专题列表或瓷片进入详情时，先触发登录。
3. 登录成功后返回原目标专题详情页。
4. 只允许访问已上架专题。

#### 页面结构

```text
顶部导航
搜索框
专题封面/头图
筛选排序与批量操作栏
商品列表
```

#### 顶部导航

| 内容 | 说明 |
| --- | --- |
| 返回按钮 | 返回上一页 |
| 标题 | 当前专题名称 |

异常状态：

| 场景 | 原型表达 |
| --- | --- |
| 专题不存在 | 页面提示：专题不存在或已下架 |
| 专题已下架 | 页面提示：专题已下架 |

#### 搜索框

| 内容 | 说明 |
| --- | --- |
| 搜索输入框 | 支持商品 ID 或商品名称模糊搜索 |
| 清空按钮 | 输入后展示，点击清空并恢复列表 |

交互要求：

1. 搜索范围仅限当前专题货盘。
2. 搜索无结果时，商品列表区展示“未找到相关商品”。
3. 搜索接口失败时，商品列表区展示加载失败和重试。

#### 专题封面/头图

展示专题创建/编辑时配置的专题封面/头图，用于匹配该专题货盘属性。

#### 筛选排序与批量操作栏

该区域是专题详情页的重点交互，请单独画清楚。

布局：

```text
┌───────────────────────────────┬──────────────┐
│ 排序项区域（横向滑动）           │ 批量添加货架   │
└───────────────────────────────┴──────────────┘
```

排序项：

| 排序项 | 说明 |
| --- | --- |
| 最新上架 | 按商品上架时间排序 |
| 7日销量 | 默认选中 |
| 佣金率 | 按佣金率排序 |
| 价格 | 按价格排序 |

交互要求：

1. 默认选中 7日销量。
2. 点击排序项刷新商品列表。
3. 排序项区域占据除右侧按钮外的最大可用宽度。
4. 排序项超出区域宽度时，左侧排序区域支持横向滑动。
5. 右侧“批量添加货架”按钮固定展示，不参与横向滑动。
6. 当前列表无商品时，批量添加货架按钮置灰。
7. 点击批量添加货架后，提示处理中，完成后展示成功数量和失败数量。
8. 批量添加货架部分失败时，提示成功数量、失败数量，暂不提供失败明细入口。
9. 批量添加货架全部失败时，Toast：批量添加失败。

#### 商品列表

商品卡片直接复用当前已封装的快手商品卡片组件，本次原型不重新设计商品卡片字段和样式。

需要在原型旁标注：

1. 商品卡片组件需透传专题 ID、来源类型、来源瓷片内容 ID。
2. 商品点击、单商品添加货架、复制 ID/口令复用现有商品卡片能力。
3. 复制 ID/口令是同一个操作，由后端接口决定返回商品 ID 还是口令，前端复制接口返回内容。
4. 从瓷片进入专题时，商品点击、添加货架、批量添加货架、复制 ID/口令均需归因到来源瓷片内容 ID。

#### 状态

| 状态 | 原型表达 |
| --- | --- |
| 专题货盘无有效商品 | 空状态：暂无商品 |
| 搜索无结果 | 空状态：未找到相关商品 |
| 商品列表接口失败 | 错误状态：加载失败，展示重试按钮 |
| 商品图片加载失败 | 展示默认商品图 |
| 添加货架失败 | Toast：添加货架失败 |
| 批量添加货架全部失败 | Toast：批量添加失败 |
| 批量添加货架部分失败 | Toast：成功 X 个，失败 Y 个 |
| 复制接口失败 | Toast：复制失败 |

## 4. 管理后台页面

### 4.1 后台-运营瓷片配置页

#### 页面目标

运营查看并配置快手选品首页的 5 个固定瓷片位。

#### 菜单路径

```text
快手选品管理 / 运营瓷片配置
```

#### 页面结构

```text
页面标题
说明提示区
瓷片位列表
分页区（如后台表格组件默认包含，可保留）
```

#### 说明提示区

提示内容：

1. 所有瓷片位均为必填，未完整配置不允许保存。
2. 配置保存后持续展示，直到运营替换内容。
3. 本期跳转类型仅支持商品专题和小程序路由，不支持 H5。

#### 瓷片位列表

表格字段：

| 字段 | 说明 |
| --- | --- |
| 瓷片位 | 主推资源位、次级资源位 1-4 |
| 封面图 | 当前配置封面；大瓷片展示排序第 1 条内容封面 |
| 内容上限 | 主推资源位最多 3 条，次级资源位固定 1 条 |
| 跳转类型 | 商品专题/小程序路由 |
| 最近更新人 | 最近操作人 |
| 最近更新时间 | 年月日时分 |
| 操作 | 编辑、点击明细 |

交互要求：

1. 页面固定展示 5 个瓷片位。
2. 不支持新增、删除瓷片位。
3. 主推资源位点击编辑，打开“主推资源位配置弹窗”。
4. 主推资源位点击点击明细，打开该资源位全部内容的点击明细。
5. 次级资源位点击编辑，打开“瓷片内容配置弹窗”。
6. 次级资源位点击点击明细，打开该次级资源位内容的点击明细。

状态：

| 状态 | 原型表达 |
| --- | --- |
| 首次未配置 | 5 行固定展示，内容列展示“待配置” |
| 列表接口失败 | 表格错误态，提供重试 |

### 4.2 弹窗-主推资源位配置

#### 弹窗目标

配置主推资源位的 1-3 条轮播内容。该弹窗形态参考“资源位配置”的数据列表样式。

#### 弹窗结构

```text
弹窗标题：主推资源位配置
顶部操作区：全部启用
数据列表
  - 内容行 1
  - 内容行 2
  - 内容行 3
底部说明
```

#### 数据列表字段

| 字段 | 说明 |
| --- | --- |
| 启用 | 控制该内容是否参与主推资源位轮播 |
| 排序 | 固定展示 1、2、3 |
| 资源图预览 | 展示该条内容封面图，未配置时展示待配置 |
| 跳转路径 | 商品专题/小程序路由 |
| 内容名称 | 该条内容名称，并展示跳转目标 |
| 创建人 | 内容创建人 |
| 创建时间 | 内容创建时间 |
| 操作 | 编辑、点击明细 |

#### 交互要求

1. 固定展示 3 行内容数据，默认启用第 1 行。
2. 可勾选启用第 2、3 行，也可点击“全部启用”一次性启用 3 行。
3. 启用内容按排序 1-3 轮播。
4. 每行点击“编辑”，打开该条内容的“瓷片内容配置弹窗”。
5. 每行点击“点击明细”，查看该条主推内容的点击明细。
6. 主推资源位列表行的“点击明细”查看该主推资源位全部内容点击明细。

状态：

| 状态 | 原型表达 |
| --- | --- |
| 第 2、3 行未启用 | 行内展示待配置，不参与轮播 |
| 启用但未配置完整 | 保存时提示补齐该行内容 |
| 点击明细查询失败 | 表格错误态，提供重试 |

### 4.3 弹窗-瓷片内容配置

#### 弹窗目标

配置单条瓷片内容。主推资源位每行内容、次级资源位内容均复用该弹窗。

#### 弹窗结构

```text
弹窗标题
瓷片位基础信息
内容配置区
底部按钮：取消、保存
```

#### 字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 内容名称 | 是 | 后台识别使用 |
| 封面图 | 是 | 根据大瓷片/小瓷片尺寸上传 |
| 跳转类型 | 是 | 商品专题、小程序路由 |
| 跳转目标 | 是 | 专题 ID/专题名称或小程序 route |
| 排序 | 主推资源位内容必填 | 决定轮播顺序 |

#### 交互要求

1. 主推资源位内容从“主推资源位配置弹窗”的单行编辑进入。
2. 次级资源位固定 1 条内容，不展示新增多条入口。
3. 主推资源位的排序由主推资源位配置弹窗中的 1-3 行顺序决定。
4. 选择商品专题时，目标选择专题 ID/专题名称，且仅允许选择已上架专题。
5. 选择小程序路由时，目标填写 route，route 必须以 `/` 开头。
6. 弹窗内不出现 H5 跳转类型。
7. 保存时校验字段完整性。

状态：

| 状态 | 原型表达 |
| --- | --- |
| 必填项缺失 | 字段下方红色校验提示，禁止保存 |
| route 格式错误 | 校验提示：route 必须以 / 开头 |
| 图片上传失败 | 上传组件失败态，允许重试 |
| 保存失败 | 保留已填内容，Toast：保存失败 |

### 4.4 弹窗-瓷片点击明细

#### 页面/弹窗目标

查看某条瓷片内容的用户点击明细。

#### 结构

```text
基础信息区
筛选区
点击明细表格
分页区
```

#### 基础信息区

展示：瓷片位、瓷片内容名称、封面、跳转类型、跳转目标。

#### 筛选区

| 筛选项 | 说明 |
| --- | --- |
| 时间范围 | 查询点击时间 |
| 用户 ID | 查询指定用户 |

#### 点击明细表格

字段：用户 ID、点击时间、瓷片位、瓷片内容、跳转类型、跳转目标。

状态：

| 状态 | 原型表达 |
| --- | --- |
| 暂无点击记录 | 空状态：暂无点击记录 |
| 查询失败 | 表格错误态，提供重试 |

### 4.5 后台-商品专题管理页

#### 页面目标

运营管理专题列表、专题分类入口、专题上下架和删除。

#### 菜单路径

```text
快手选品管理 / 商品专题管理
```

#### 页面结构

```text
页面标题
筛选区
顶部操作区
专题列表
分页区
```

#### 筛选区

## 7. 原型标注规范

请在墨刀原型中对以下内容增加文字标注，便于研发理解：

1. 瓷片区：所有瓷片位后台必填；保存后持续展示，无生效/失效时间。
2. 主推资源位：编辑时先进入 3 行内容数据列表，默认启用 1 行，可全部启用，启用内容按排序轮播。
3. 小瓷片：每个位置固定 1 条内容。
4. 瓷片跳转：本期仅商品专题、小程序路由，不支持 H5。
5. 专题入口：本需求不新增金刚区入口，不改造 Banner 功能；运营可复用现有 Banner 配置跳转专题列表。
6. 专题分类：作为筛选项；默认不选中；单选；再次点击取消选中。
7. 专题详情：需要登录；未登录进入详情前先登录。
8. 专题详情排序：最新上架、7日销量、佣金率、价格；默认 7日销量。
9. 批量添加货架：与排序区同一行，右侧固定按钮；左侧排序区可横向滑动。
10. 商品卡片：复用现有快手商品卡片组件，不重画字段逻辑。
11. 复制 ID/口令：同一个操作，由后端决定返回商品 ID 或口令。
12. 来源归因：从瓷片进入专题后，专题访问和专题商品行为均携带来源瓷片内容 ID。
13. 选择商品计划：专题新增/编辑页使用浅灰底配置容器，包含人工添加商品计划和自动匹配商品计划两个 Tab。
