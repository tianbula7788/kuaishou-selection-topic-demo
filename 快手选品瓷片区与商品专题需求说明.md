# 快手选品瓷片区与商品专题需求说明

## 1. 需求背景

当前快手选品首页已有搜索、金刚区、商品列表和筛选能力，但缺少可灵活配置的运营资源位，也缺少围绕活动、品类、节点搭建商品专题的能力。

本次需求希望在快手选品中补充两类能力：

1. 首页新增运营瓷片区，承接主推活动、专题或小程序内部页面。
2. 新增商品专题，支持运营组建专题货盘，并统计专题访问和商品转化行为。

## 2. 需求目标

1. 快手选品首页金刚区下方新增“1 大 + 4 小”运营瓷片区。
2. 管理后台支持配置每个瓷片位的封面、跳转目标和排序；所有瓷片位必须配置内容后才允许保存。
3. 快手选品新增商品专题列表页和专题详情页；专题列表页可由运营复用现有 Banner 配置能力跳转进入，本需求不改造 Banner 功能。
4. 管理后台支持专题分类、专题内容和专题货盘维护。
5. 支持记录瓷片点击、专题访问、专题商品行为，并能将专题商品行为归因到来源瓷片。

## 3. 本期范围

### 3.1 本期做

| 模块 | 范围 |
| --- | --- |
| 首页瓷片区 | 固定 1 个大瓷片 + 4 个小瓷片 |
| 大瓷片 | 最多配置 3 条内容，按后台排序顺序轮播 |
| 小瓷片 | 每个小瓷片位必须配置 1 条内容 |
| 瓷片跳转 | 支持跳转商品专题、小程序内部路由 |
| 专题列表 | 未登录用户可浏览 |
| 专题详情 | 进入专题详情页需要登录 |
| 专题分类 | 作为专题聚合标签使用，专题可不关联分类 |
| 专题货盘 | 商品来自竞佣成功的快手选品池 |
| 货盘组建 | 支持人工添加商品计划、自动匹配商品计划；人工添加下支持手动选择商品和 Excel 导入商品 ID 匹配商品库；自动匹配保存为动态规则货盘 |
| 埋点 | 记录瓷片点击、专题访问、专题商品行为和来源归因 |

### 3.2 本期不做

| 内容 | 说明 |
| --- | --- |
| H5 跳转 | 瓷片区本期不支持 H5 |
| 个性化投放 | 不做不同用户展示不同瓷片内容 |
| 动态瓷片布局 | 不做后台自定义瓷片数量或布局 |
| 专题多分类 | 一个专题最多关联一个分类，不做多分类挂载 |
| Banner 功能 | 不做任何 Banner 功能改动、配置能力改造或优化 |
| 复杂数据看板 | 本期以明细记录和基础统计口径为主 |
| 审批流 | 瓷片配置和专题上下架不做审批 |

## 4. 页面原型与交互说明

本需求涉及两个系统：小程序和管理后台。页面说明按“系统 → 界面 → 布局 → 板块 → 内容项 → 逻辑交互 → 空/异常状态”的结构描述。

### 4.1 系统与界面总览

| 系统 | 界面 | 页面目标 | 主要使用者 |
| --- | --- | --- | --- |
| 小程序 | 快手选品首页 | 在现有首页中增加运营瓷片区 | 用户 |
| 小程序 | 商品专题列表页 | 展示专题集合，支持按专题分类聚合浏览 | 用户 |
| 小程序 | 商品专题详情页 | 展示某个专题下的商品货盘，并支持搜索、排序和商品操作 | 用户 |
| 管理后台 | 运营瓷片配置页 | 管理首页 1 大 + 4 小瓷片区内容 | 运营 |
| 管理后台 | 主推资源位配置弹窗 | 管理主推资源位 1-3 条轮播内容 | 运营 |
| 管理后台 | 瓷片内容配置弹窗 | 配置单条瓷片内容的封面和跳转目标 | 运营 |
| 管理后台 | 瓷片点击明细页/弹窗 | 查看某条瓷片内容的用户点击明细 | 运营、数据人员 |
| 管理后台 | 商品专题管理页 | 管理专题列表、专题上下架、专题货盘 | 运营 |
| 管理后台 | 专题新增/编辑页 | 编辑专题基础信息和专题货盘 | 运营 |
| 管理后台 | 专题分类管理抽屉 | 在商品专题管理页内维护专题分类 | 运营 |

## 5. 小程序页面说明

### 5.1 快手选品首页

#### 5.1.1 页面整体结构

快手选品首页在现有结构上新增运营瓷片区。

页面结构：

```text
顶部搜索/现有首页头部
金刚区
  - 原有金刚区入口
运营瓷片区
商品分类 Tab
筛选区
商品列表
底部导航
```

#### 5.1.2 运营瓷片区

布局结构：

```text
┌──────────────┐  ┌──────────┐ ┌──────────┐
│              │  │ 小瓷片 1  │ │ 小瓷片 2  │
│   大瓷片      │  └──────────┘ └──────────┘
│              │  ┌──────────┐ ┌──────────┐
│              │  │ 小瓷片 3  │ │ 小瓷片  4 │
└──────────────┘  └──────────┘ └──────────┘
```

板块内容：

| 瓷片位 | 类型 | 位置 | 内容规则 |
| --- | --- | --- | --- |
| 主推资源位 | 大瓷片 | 左侧 | 至少 1 条、最多 3 条内容，按后台排序轮播 |
| 次级资源位 1 | 小瓷片 | 右上左 | 必须 1 条内容 |
| 次级资源位 2 | 小瓷片 | 右上右 | 必须 1 条内容 |
| 次级资源位 3 | 小瓷片 | 右下左 | 必须 1 条内容 |
| 次级资源位 4 | 小瓷片 | 右下右 | 必须 1 条内容 |

内容项：

| 内容项 | 说明 |
| --- | --- |
| 封面图 | 后台配置；大瓷片、小瓷片按不同尺寸展示 |
| 跳转类型 | 商品专题、小程序路由 |
| 跳转目标 | 专题 ID 或小程序 route |

交互逻辑：

1. 大瓷片存在多条内容时，按后台排序顺序轮播。
2. 小瓷片固定展示对应瓷片位配置的 1 条内容，不轮播。
3. 点击瓷片时，先记录瓷片点击埋点，再执行跳转。
4. 跳转类型为商品专题时，进入专题详情页；如果用户未登录，先触发登录，登录成功后进入目标专题详情页。
5. 跳转类型为小程序路由时，跳转指定内部页面。
6. 用户从瓷片进入专题详情后，专题商品行为需要保留来源瓷片归因。

空状态：

| 场景 | 展示/处理 |
| --- | --- |
| 后台未完整配置瓷片内容 | 后台不允许保存，因此小程序正常情况下不出现缺失瓷片位 |
| 接口返回空内容 | 小程序端隐藏瓷片区，并上报/记录异常，避免阻塞首页主流程 |
| 某个封面图加载失败 | 当前瓷片位展示默认占位图，点击逻辑仍按配置执行 |

异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 瓷片配置接口失败 | 隐藏瓷片区，首页其他内容正常展示 |
| 瓷片跳转专题已下架或不存在 | 提示“内容已失效”或“专题已下架” |
| 小程序路由无效 | 提示“页面暂不可用” |

#### 5.1.3 专题列表入口说明

本需求专题列表页入口由运营人员复用现有 Banner 配置能力完成，配置跳转目标为商品专题列表页的小程序路由即可。

### 5.2 商品专题列表页

#### 5.2.1 页面目标

展示可访问的商品专题集合，支持用户通过专题分类快速浏览不同专题。

访问规则：

1. 未登录用户允许浏览专题列表页。
2. 专题列表页只展示已上架专题。
3. 专题分类未配置时，隐藏分类区域。

#### 5.2.2 页面整体结构

```text
顶部导航
专题头图
专题分类区域（有分类时展示）
专题列表区域
  - 专题卡片 1
  - 专题卡片 2
  - ...
```

#### 5.2.3 顶部导航板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 返回按钮 | 返回上一页 |
| 页面标题 | 商品专题 |

异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 页面从外部入口进入 | 返回快手选品首页或上一页 |
| 页面从分享或外部路径进入 | 返回上一页；无上一页时返回首页 |

#### 5.2.4 专题头图板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 专题头图 | 展示专题创建/编辑时配置的专题封面，用于表达该专题货盘属性 |

空/异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 专题未配置头图 | 专题上架前需配置专题封面；正常情况下不出现空头图 |
| 专题头图加载失败 | 展示默认专题头图 |

#### 5.2.5 专题分类板块

展示条件：

1. 后台配置了至少 1 个专题分类时展示。
2. 后台未配置任何专题分类时隐藏整个分类区域。

内容项：

| 内容项 | 说明 |
| --- | --- |
| 分类名称 | 展示后台配置的专题分类名称 |
| 分类顺序 | 按后台拖拽排序后的顺序正序展示 |

交互逻辑：

1. 专题分类区域作为专题列表的筛选项使用。
2. 默认不选中任何专题分类，专题列表展示全部已上架专题。
3. 专题分类为单选，点击某个分类后，专题列表切换为该分类下的专题。
4. 已选中某个分类时，再次点击该分类则取消选中，专题列表恢复展示全部已上架专题。
5. 未关联分类的专题在默认未筛选状态下正常展示；选中某个分类后，仅展示该分类下的专题。

空/异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 无专题分类 | 隐藏专题分类区域 |
| 当前分类下无专题 | 专题列表区域展示空状态 |
| 分类接口失败 | 隐藏分类区域，专题列表可继续展示 |

#### 5.2.6 专题列表板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 专题卡片 | 展示专题名称、专题副标题、专题封面、商品预览 |
| 商品预览 | 展示专题内佣金率倒序top3的商品的商品主图、商品价格、商品佣金率、商品佣金金额 |

交互逻辑：

1. 点击专题卡片时，若用户已登录，进入专题详情页。
2. 点击专题卡片时，若用户未登录，触发登录；登录成功后进入原目标专题详情页。
3. 专题卡片只展示已上架专题。
4. 专题列表按后台专题排序展示；同一排序下按更新时间倒序展示，具体实现可由研发确认。

空状态：

| 场景 | 展示/处理 |
| --- | --- |
| 全部无已上架专题 | 展示“暂无专题”空状态 |
| 当前分类下无专题 | 展示“当前分类暂无专题”空状态 |

异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 专题列表接口失败 | 展示加载失败状态，下拉界面可刷新重新加载 |
| 专题封面加载失败 | 专题卡片展示默认封面 |
| 商品预览为空 | 专题卡片仅展示专题基础信息，不展示商品预览区，按理来讲，不会出现商品预览为空的情况 |

### 5.3 商品专题详情页

#### 5.3.1 页面目标

展示某个专题下的商品货盘，支持用户在专题内搜索、排序、点击商品、批量添加货架。

访问规则：

1. 专题详情页需要登录后访问。
2. 未登录用户从专题列表或瓷片进入详情页时，先触发登录，登录成功后回到目标专题详情页。
3. 仅已上架专题允许访问。

#### 5.3.2 页面整体结构

```text
顶部导航
专题搜索框
专题封面
筛选排序与批量操作栏
商品列表区
```

#### 5.3.3 顶部导航板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 返回按钮 | 返回上一页 |
| 页面标题 | 专题名称 |

异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 专题不存在 | 展示“专题不存在或已下架” |
| 专题已下架 | 展示“专题已下架” |

#### 5.3.4 搜索框板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 搜索输入框 | 支持商品 ID 或商品名称模糊搜索 |
| 清空按钮 | 输入内容后展示，点击清空搜索词 |

交互逻辑：

1. 搜索范围仅限当前专题货盘。
2. 输入商品 ID 或商品名称后触发搜索，具体触发方式可按现有搜索组件规则处理。
3. 清空搜索词后恢复当前专题默认商品列表。

空/异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 搜索无结果 | 商品列表区展示“未找到相关商品” |
| 搜索接口失败 | 商品列表区展示加载失败状态，提供重试 |

#### 5.3.5 专题封面板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 专题封面图 | 展示后台创建/编辑专题时配置的专题封面/头图 |

异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 封面加载失败 | 展示默认专题封面 |

#### 5.3.6 筛选排序与批量操作栏

该板块位于专题封面下方、商品列表上方，筛选排序区域与批量操作按钮在同一行内展示。

布局结构：

```text
┌───────────────────────────────┬──────────────┐
│ 筛选排序区域（可横向滑动）        │ 批量添加货架   │
└───────────────────────────────┴──────────────┘
```

筛选排序项：

| 排序项 | 说明 |
| --- | --- |
| 最新上架 | 按商品上架时间排序 |
| 7日销量 | 默认排序项，按 7 日销量排序 |
| 佣金率 | 按佣金率排序 |
| 价格 | 按价格排序 |

批量操作：

| 操作 | 说明 |
| --- | --- |
| 批量添加货架 | 将当前专题商品列表范围内的商品批量添加货架 |

交互逻辑：

1. 默认选中 7日销量排序。
2. 点击排序项后刷新商品列表。
3. 支持升序/降序的排序项是否提供切换态，由设计和现有商品列表组件能力确认；如现有商品列表已有规则，优先复用。
4. 筛选排序区域和批量添加货架按钮保持同一行展示。
5. 批量添加货架按钮固定在该行右侧，不参与横向滑动。
6. 当筛选排序项内容较多、超出筛选排序区域可展示宽度时，筛选排序区域在可用最大宽度内支持左右滑动，以展示更多筛选排序项。
7. 点击批量添加货架后，触发批量添加货架流程。
8. 批量添加货架的商品范围默认为当前专题列表在当前搜索词和排序条件下命中的商品集合；如后端或现有能力只支持当前分页商品，则需在技术方案中明确限制。
9. 批量添加货架完成后，需要提示成功数量和失败数量；如存在失败商品，暂不展示失败原因或提供失败明细入口。

异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 排序接口失败 | 保留当前列表，提示加载失败或在列表区展示失败状态 |
| 当前列表无商品 | 批量添加货架按钮置灰或不可点击 |
| 批量添加货架全部失败 | 提示批量添加失败，暂不展示失败原因 |
| 批量添加货架部分失败 | 提示成功数量、失败数量，暂不提供失败明细 |

#### 5.3.7 商品列表板块

内容项：

| 内容项 | 说明 |
| --- | --- |
| 快手商品卡片 | 直接复用当前已封装的快手商品卡片组件，本需求不重新定义商品卡片样式和字段 |
| 专题上下文参数 | 调用商品卡片组件时需透传专题 ID、来源类型、来源瓷片内容 ID 等参数，用于行为归因 |


交互逻辑：

1. 商品展示、商品点击、单商品添加货架、复制 ID/口令等基础能力复用现有快手商品卡片组件能力。
2. 点击商品卡片，进入现有商品详情或商品链路，并记录专题商品点击行为。
3. 点击添加货架，复用现有快手商品卡片中的添加货架能力，并记录专题添加货架行为。
4. 点击复制 ID/口令，复用现有快手商品卡片中的复制能力；前端调用后端接口，后端根据商品情况返回商品 ID 或口令，前端复制接口返回内容，并记录复制 ID/口令行为。
5. 点击批量添加货架，执行当前专题商品集合的批量添加货架能力，并记录专题批量添加货架行为。

商品计划配置方式：

| 方式 | 页面交互 |
| --- | --- |
| 人工添加商品计划 | 默认选中；展示“选择活动计划”标题、“添加商品计划”按钮和已选商品计划卡片 |

1. 点击“添加商品计划”，打开商品选择弹窗。

自动匹配商品计划交互：

2. 配置区展示“当前符合条件商品计划 X 个”的统计入口。
3. 支持新增计划规则，规则内可配置多个条件。
4. 条件支持商品类目、佣金率、价格、7日销量、快手官方商品标签。
5. 商品类目和快手官方商品标签支持多选；区间类条件支持最小值和最大值。
6. 点击“添加条件”新增规则条件；点击“重置”清空当前规则。
7. 自动匹配规则确认后，保存为动态规则货盘；后续快手选品池每天新增入库商品时，符合条件的新商品需要自动进入该专题货盘。

| 字段 | 说明 |

| 7日销量 | 区间选择 |
| 商品类目 | 多选 |
| 快手官方商品标签 | 多选，枚举包含：平台补贴、已获得流量扶持、直播返佣、直播同款、政策补贴 |

3. 重复商品 ID 自动去重。

| Excel 部分匹配失败 | 成功商品进入货盘，失败商品展示失败明细 |

4. 用户点击分类行左侧排序按钮，可上下拖动调整分类顺序。
5. 正常展示态下，鼠标悬浮到分类行后展示“编辑”和“删除”按钮。
6. 点击“编辑”后，该行进入编辑态，可修改分类名称。
7. 点击“删除”后弹出二次确认弹窗。
8. 删除确认弹窗需提示该分类当前关联的专题数量，例如“该专题分类已关联 X 个专题，确认要删除吗？”
9. 用户点击确认后删除该分类；已关联该分类的专题统一置为未分类。

空/异常状态：

| 场景 | 展示/处理 |
| --- | --- |
| 无专题分类 | 抽屉内展示“暂无专题分类”，保留新增按钮 |
| 新增分类名称为空 | 禁止确定，提示填写分类名称 |
| 分类名称重复 | 不允许重复，吐司提示：分类名称重复 |
| 删除失败 | 保留当前分类，展示失败提示 |
| 排序保存失败 | 恢复原排序或提示刷新后重试 |

### 7.1 瓷片点击埋点

触发时机：用户点击任一瓷片。

记录字段：

| 字段 | 说明 |
| --- | --- |
| 用户 ID | 仅记录已登录用户；匿名/游客用户点击瓷片不做记录 |
| 点击时间 | 用户点击时间 |
| 瓷片位 | 主推资源位/次级资源位 |
| 瓷片内容 ID | 被点击的瓷片内容 |
| 瓷片内容名称 | 后台配置名称 |
| 跳转类型 | 商品专题/小程序路由 |
| 跳转目标 | 专题 ID 或 route |

后台需支持查看单条瓷片内容的点击明细。

### 7.2 专题访问埋点

触发时机：用户进入专题详情页。

记录规则：

1. 一天内，同一用户访问同一专题只记录一次。
2. 去重口径为：日期 + 用户 ID + 专题 ID。

记录字段：

| 字段 | 说明 |
| --- | --- |
| 日期 | 访问日期 |
| 用户 ID | 访问用户 |
| 专题 ID | 被访问专题 |
| 专题名称 | 访问时专题名称 |
| 分类 ID | 专题所属分类 |
| 首次访问时间 | 当天首次进入专题详情页时间 |
| 来源类型 | 通用 Banner/瓷片区/分享/其他 |
| 来源瓷片内容 ID | 来源为瓷片区时记录 |

### 7.3 专题商品行为埋点

触发行为：

1. 商品点击。
2. 添加货架。
3. 批量添加货架。
4. 复制 ID/口令。

记录字段：

| 字段 | 说明 |
| --- | --- |
| 用户 ID | 操作用户 |
| 行为时间 | 操作发生时间 |
| 行为类型 | 商品点击/添加货架/批量添加货架/复制 ID/口令 |
| 专题 ID | 行为发生的专题 |
| 商品 ID | 单商品行为记录被操作商品；批量添加货架可记录商品 ID 列表或由批量任务明细承接 |
| 商品名称 | 单商品行为记录被操作商品名称 |
| 商品位置 | 单商品行为记录商品在专题列表中的位置 |
| 批量操作商品范围 | 仅批量添加货架行为记录，说明本次批量操作覆盖的商品范围 |
| 批量成功数量 | 仅批量添加货架行为记录 |
| 批量失败数量 | 仅批量添加货架行为记录 |
| 复制结果类型 | 仅复制 ID/口令行为记录，取值为商品 ID 或口令 |
| 来源类型 | 通用 Banner/瓷片区/分享/其他 |
| 来源瓷片内容 ID | 来源为瓷片区时记录 |

## 8. 异常与边界

| 场景 | 处理 |
| --- | --- |
| 后台瓷片内容未配置完整 | 不允许保存，提示补齐所有瓷片位内容 |
| 未登录点击专题列表卡片 | 触发登录，登录成功后进入专题详情 |
| 专题已下架或不存在 | 提示专题已下架或不存在 |
| 专题无有效商品 | 专题详情展示空状态 |
| 瓷片跳转专题已不可用 | 提示内容已失效 |
| Excel 商品 ID 匹配失败 | 返回失败明细和原因 |
| 批量添加货架部分失败 | 提示成功数量和失败数量，可查看失败原因 |
| 埋点上报失败 | 不阻塞用户主流程 |

## 9. 验收标准

### 9.1 瓷片区

1. 首页金刚区下方可展示 1 大 + 4 小瓷片区。
2. 大瓷片最多 3 条内容并按排序轮播。
3. 小瓷片每个位置必须展示 1 条内容。
4. 瓷片支持跳转商品专题和小程序内部路由。
5. 后台保存瓷片配置时，必须校验所有瓷片位均已配置内容。
6. 后台可查看瓷片点击明细。
7. 已登录用户点击瓷片时正确记录点击埋点，匿名/游客用户点击瓷片不记录。

### 9.2 商品专题

1. 不在快手首页金刚区新增专题入口，且不做任何 Banner 功能改动；运营可复用现有 Banner 配置能力跳转专题列表页。
2. 未登录用户可浏览专题列表。
3. 未登录用户进入专题详情前需登录。
4. 专题列表仅展示已上架专题；未配置专题分类时隐藏分类区域。
5. 专题详情页展示专题封面、搜索、筛选排序与批量操作栏、商品列表，默认按 7日销量排序。
6. 专题详情页筛选排序区域与批量添加货架按钮在同一行；筛选排序项超出可展示宽度时，筛选排序区域可左右滑动，批量添加货架按钮固定在右侧。
7. 后台可在商品专题管理页面通过抽屉管理专题分类，并可管理专题基础信息和专题货盘。
8. 专题货盘仅允许使用竞佣成功的快手选品池商品。
9. 自动匹配商品计划保存为动态规则货盘，新增入库且符合条件的商品可进入专题。
10. Excel 导入商品 ID 时可返回成功和失败结果。

### 9.3 埋点与归因

1. 已登录用户瓷片点击记录包含用户、时间、瓷片位、瓷片内容和跳转目标；匿名/游客用户不记录瓷片点击。
2. 专题访问按“一天一个用户一个专题一次”去重。
3. 专题内商品点击、添加货架、批量添加货架、复制 ID/口令均能记录。
4. 从瓷片进入专题后的商品行为能记录来源瓷片内容 ID。
